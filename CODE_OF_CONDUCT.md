# Code Of Conduct

Funplay should be a practical, respectful place to build ambitious developer tools.

## Expected Behavior

- Be direct, kind, and constructive.
- Assume good intent while still naming concrete problems.
- Keep feedback focused on code, design, documentation, behavior, and user impact.
- Respect different experience levels, backgrounds, tools, and languages.

## Unacceptable Behavior

- Harassment, threats, or personal attacks
- Sexualized or discriminatory language
- Publishing private information without permission
- Sustained disruption of issues, discussions, or pull requests
- Bad-faith security reports or exploit publication before maintainers can respond

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject pull requests; or restrict participation when behavior harms the project or its contributors.

If you see a problem, contact the maintainers with context and links. If no private contact is listed yet, open a minimal public issue asking for a maintainer contact without sharing private details.
